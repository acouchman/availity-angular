Using UI-Router Continued

availity.demo.config(function($stateProvider, $urlRouterProvider){ //code }

The default config must reference $stateProvider and $urlRouterProvider to be used. In order for UI-Router to kick in a ui-view must be present in the markup, if no ui-view is present no routing will take place.

$urlRouterProvider.otherwise('/Wizard-1'); $stateProvider .state('Wizard-1', { url: '/Wizard-1', templateUrl: 'templates/wizard-1-template.html' }) .state('Wizard-2', { url: '/Wizard-2', templateUrl: 'templates/wizard-2-template.html' }) .state('Wizard-3', { url: '/Wizard-3', templateUrl: 'templates/wizard-3-template.html' });

$urlrouterProvider.otherwise() marks the default state if no state is given, this is only used if ui-view is present. The state provider gives a list of states along with their intended URL path. The first parameter int .state states the name of the state, url: states what the url for the state is, and templateUrl: gives what view will be loaded into the ui-view in a get request.
